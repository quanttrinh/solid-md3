# Product

## Register

product

## Platform

web

## Users

SolidJS developers evaluating or using the `@quantt/solid-md3` component library. They arrive to explore components, copy code, and see how each piece works in practice.

## Product Purpose

An interactive developer playground showcasing every component in the `@quantt/solid-md3` library. Each component is displayed with live examples — different variants, sizes, states — so developers can understand the API, see the visual result, and grab working code. The playground is the single source of truth for what the library looks like when rendered.

## Positioning

The definitive showcase for `@quantt/solid-md3` — every component, every variant, every state, all in one interactive page.

## Brand Personality

Precise, compliant, clean. The demo is a mirror of the library's values: MD3-spec components displayed with clarity, no decoration beyond what the components themselves provide.

## Anti-references

Over-styled component galleries with unnecessary animations or decorative layouts. The playground should feel like a working tool, not a marketing page. No hero sections, no testimonials, no pricing — just components, clearly presented.

## Design Principles

1. **Components speak for themselves.** The playground exists to showcase the library, not to impress with its own design. Every visual decision serves component clarity.
2. **Interactive by default.** Live, working examples beat static screenshots. Every component should be explorable — toggle states, change props, see results.
3. **Copy-ready code.** Developers arrive to grab code. Every example should be easy to extract and drop into a real app.
4. **Complete coverage.** If it's in the library, it's in the playground. No missing components, no half-documented APIs.

## Accessibility & Inclusion

WCAG AAA compliance. The playground itself must be keyboard-navigable, screen-reader friendly, and accessible — a showcase for an accessible library must be accessible. Respects `prefers-reduced-motion`. Navigation between component sections is structured and logical.
