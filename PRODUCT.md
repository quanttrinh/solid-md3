# Product

## Register

product

## Platform

web

## Users

SolidJS developers building production applications. Primarily internal use — you and your apps. The library powers your own projects; it is not published as a public npm package.

## Product Purpose

A comprehensive SolidJS component library implementing Material Design 3 specifications. Every component follows Google's MD3 guidelines for color, typography, motion, shape, and elevation. The library provides a complete set of UI primitives — buttons, inputs, navigation, containers, indicators, messaging — so you can build app interfaces that are spec-accurate, accessible, and consistent without reinventing standard patterns.

## Positioning

The most complete, spec-accurate Material Design 3 implementation for SolidJS — built on Ark UI primitives with Tailwind CSS v4 tokens, following Google's guidelines precisely.

## Brand Personality

Precise, compliant, clean. The voice of a well-tuned spec implementation: nothing extra, nothing missing. Confident through accuracy, not decoration.

## Anti-references

Over-styled component libraries with heavy animations, gradients, decorative motion, or invented affordances. Generic Material kits without framework soul. Minimal or unfinished feeling components that lack polish. The library should feel like MD3 rendered through SolidJS, not a design exercise.

## Design Principles

1. **Spec is the source.** Google's MD3 guidelines are the reference implementation. When in doubt, defer to the spec — color roles, type scale, elevation levels, motion curves, shape tokens.
2. **Composable over monolithic.** Ark UI provides headless primitives; SolidJS provides reactivity. Compose from clean building blocks, don't ship pre-baked layouts.
3. **Complete state coverage.** Every interactive component ships with default, hover, focus, active, disabled, loading, and error states. No half-states.
4. **Token-driven.** All visual decisions flow from MD3 design tokens (CSS custom properties). Themeable, overridable, consistent.
5. **Accessibility is not optional.** WCAG AAA compliance across all components. Keyboard navigation, screen reader support, proper ARIA attributes — built in, not bolted on.

## Accessibility & Inclusion

WCAG AAA compliance target. Follows Material Design 3 accessibility guidelines: minimum touch targets, proper contrast ratios, keyboard navigation patterns, screen reader semantics. Every interactive component includes focus-visible states, aria labels, and proper role attributes. Reduced motion respects `prefers-reduced-motion`.
